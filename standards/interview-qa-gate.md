# Interview QA Gate（面接稿 出荷前検査）

Last updated: 2026-06-16

## これは何か

面接稿（`<company>_interview_integrated_<date>.md`）を「完成」と呼ぶ前に必ず通す**強制検査**。1項目でもFailなら修正して再検査。検査結果は基本ファイル末尾の「QAゲート記録」に残す（面接稿には書かない）。

> 原則: 検査は自己申告ではなく**実測**で行う。長さは文字数を数え、文末は分布を数え、会社入れ替えテストは実際に社名を置換して読む。

> **独立検証の原則（重要）**: 書いた本人（同一AIセッション）が自分の原稿を採点すると、見落としと身贔屓が必ず出る。大手企業・二次以降・最終面接など重要度の高い面接稿は、**原稿を書いたセッションとは別の検証サブエージェントに Gate A〜D を実測させる**（手順は末尾「独立検証サブエージェントの回し方」）。検証者には「合格させること」ではなく「Fail を見つけること」を仕事として与える。軽い一次面接の基礎質問のみの稿は自己検査でよいが、その判断自体を記録する。

## Gate A: 事実・整合（嘘と矛盾の根絶）

| # | チェック | 方法 |
| --- | --- | --- |
| A1 | 全エピソード・経歴・日付が `candidate-profile.md` と一致 | 突き合わせ |
| A2 | 使用した数字がすべて `evidence-ledger.md` に存在し、Status相応の表現（Needs Check/Weakは控えめ表現） | 数字を列挙して照合 |
| A3 | Packaging Warnings を犯していない（禁止項目は `candidate-profile.md` 末尾の Packaging Warnings を正本とする。例: 未完成物を完成品と言う / 補助経験を主役級に誇張する / 非公開物を公開前提にする 等） | 該当回答を精読 |
| A4 | 提出済みES・履歴書・職務経歴書の記載と矛盾しない（面接は提出書類に基づいて聞かれる） | 同社フォルダの提出書類と照合 |
| A5 | 会社名・職種名・サービス名・面接官所属の表記が正確（旧社名・誤読に注意） | 公式サイトで確認 |
| A6 | 個人成果/チーム成果/プロジェクト全体の区別が回答文面から読み取れる | 数字を含む回答を精読 |
| A7 | `[仮説]` `[情報不足→逆質問TODO]` タグが面接稿に残っていない（基本ファイル側へ） | grep |
| A8 | 他社面接稿からの流用文がそのまま残っていない（社名だけ替えた痕跡） | 直近の他社統合版と比較 |

## Gate B: 構造（聞かれたことに、短く、深掘りを誘って答える）

| # | チェック | 基準 |
| --- | --- | --- |
| B1 | 全回答が結論ファースト | 第一文（50字以内）が質問への直接の答え |
| B2 | 長さが基準表に準拠 | delivery-standard §1。**全回答の文字数を実測**し、90秒/450字超ゼロ |
| B3 | ワンメッセージ | 1回答に主張1つ。強み・理由の3連並列がない |
| B4 | フック設計 | 主要4答（志望動機・自己PR・ガクチカ・前職）に深掘りを誘う引きが1つ以上 |
| B5 | 深掘り2層の防御 | 主要回答すべてに追撃Q&A（15〜30秒の返し）が用意され、基本ファイルの深掘りツリーと一致 |
| B6 | 志望動機が4層接続 | やりたいこと→業界→職種→会社の流れが話し言葉の中で自然に通る |
| B7 | 逆質問 | 5問以上、3階層（事業/役割/成長）+ 段階適合（一次二次=現場型、最終=経営型）、仮説検証型を1問以上 |
| B8 | ▼キーワード行 | 全回答に3〜4語のキーワード行がある |

## Gate C: 日本語（ネイティブに自然な話し言葉）

| # | チェック | 基準 |
| --- | --- | --- |
| C1 | 敬語NGゼロ | delivery-standard §3 のNG表と突き合わせ。二重敬語・バイト敬語・了解しました等が0件 |
| C2 | 書き言葉の残存ゼロ | §4変換表（実施/および/である/第一に/把握 等）をgrep的に検査 |
| C3 | 自然度Lint 12項目クリア | §5。特に: 文頭「私は/私の」3割以下、同一文末3連続なし、「〜たいです」回答内3回未満 |
| C4 | 文の長さ | **読点数と文字数の両方を実測する**（片方だけは不可）。①読点4個以上の文ゼロ ②1文50字超を列挙し、原則ゼロ・最長でも55字以内。※「読点だけ数えて文長を見落とす」のが自己検査の頻発バグ（過去に実例あり。50字超が約30文残ったまま読点数のみでPass判定していた） |
| C5 | 結びの多様性 | 「貢献したいです」系の結びが全回答の半分以下 |
| C6 | 誤字・変換ミス・不自然な漢字（ひらくべき語: 是非→ぜひ、頂く→いただく、出来る→できる、致します→いたします） | 精読 |
| C7 | 音読可能性 | 舌が回らない語・息継ぎ不能な文がない（頭の中で発話速度で再生） |
| C8 | 中国語話者翻訳調の検査 | §5後半（〜の方面、抱負表現の連発、アナロジー欠落）が0件 |
| C9 | 声音層の準備 | `knowledge/native-delivery-pronunciation.md` に基づき、基本ファイルに**発音リスクリスト**がある。面接で必ず口にする社名・術語・本人氏名の読みが（OJAD/辞典で）確認済みか「要確認」明記。未確認の語をキーワード行に含めていない |

## Gate D: 戦略（この会社で勝つ原稿か）

| # | チェック | 基準 |
| --- | --- | --- |
| D1 | 求める人物像マッピング | 企業の人物像キーワードそれぞれに対応する回答が存在（対応表は基本ファイル側） |
| D2 | 差別化要素 | 主要回答に、本人の差別化トークン（`candidate-profile.md` の Core Positioning）が1つ以上。かつ同業他社受験者が言えない一文がある |
| D3 | **会社入れ替えテスト** | 志望動機・なぜ当社の社名を競合他社名に置換して読む。**そのまま成立したらFail**（Layer 4が汎用ということ） |
| D4 | 段階適合 | 面接段階（一次/二次/最終）に対し重点質問・逆質問の視座が合っている。最終なら入社意思・覚悟系が厚い |
| D5 | リスク防御カバー | candidate-profile の標準リスク（本人の Interview Risk Defense に定義: 例 ブランク・新卒枠・過去の失敗・数字・ビザ・専門深掘り）＋企業特有リスクすべてに回答が存在 |
| D6 | 情報の鮮度 | 企業情報（ニュース・IR・採用ページ）を面接7日前以内に再確認した。古い事実（終了したサービス等）を志望動機に使っていない |

## QAゲート記録の様式（基本ファイル末尾に貼る）

```markdown
## QAゲート記録

検査日: YYYY-MM-DD / 検査者: (AI session)
| Gate | 結果 | Fail項目と修正内容 |
| --- | --- | --- |
| A 事実・整合 | Pass |  |
| B 構造 | Pass | B2: 志望動機480字→400字に短縮 |
| C 日本語 | Pass | C3: 文末「です」3連続を2箇所修正 |
| D 戦略 | Pass | D3: Layer4に中計の固有施策を追加して再テスト |
実測メモ: 最長回答=志望動機398字(80秒) / 「貢献したい」結び 3/22回答 / 文頭「私」 5/22回答
```

## 独立検証サブエージェントの回し方（大手・二次以降・最終で必須）

原稿を書いたセッションが、別のサブエージェント（Task / Agent）を立てて検証させる。目的は**粗探し**であって追認ではない。

1. 検証エージェントに渡すもの: 面接稿（integrated）、基本ファイル（QAゲート記録欄を空で）、`candidate-profile.md`、`evidence-ledger.md`、本 QA Gate、`standards/japanese-delivery-standard.md`。
2. 指示文の骨子: 「あなたは面接稿の独立検査官。合格させるのが仕事ではなく、Gate A〜Dの**Fail を実測で見つける**のが仕事。①全回答の文字数を数えて90秒/450字超を列挙 ②文末3連続・文頭『私』比率・『貢献したい』結びの回数を数える ③敬語NG表・書き言葉残存をgrep的に列挙 ④**会社入れ替えテスト**を実際に社名置換して実行し、成立してしまう回答を挙げる ⑤candidate-profileとの事実矛盾・Packaging Warnings 4禁の違反を挙げる。Pass/Failと根拠（数値・該当文）を表で返す。曖昧な合格判定は禁止」。
3. 返ってきた Fail を修正 → 必要なら再検証。結果と実測メモを基本ファイルの「QAゲート記録」に貼る（検証者＝別エージェントである旨も書く）。
4. 自己検査で済ませた場合（軽い一次のみ等）は、その判断理由を1行記録する。

## 納品時の練習指示（毎回ユーザーに添える）

1. 各回答を**音読3回**（1回目: 読む / 2回目: キーワード行だけ見て話す / 3回目: 何も見ずに話す）
2. **1回は録音して聞き返す**（速すぎ・語尾伸び・棒読み・**句末上がり**チェック。中国語母語の癖は語尾の上げに出やすい→`knowledge/native-delivery-pronunciation.md` §4）
3. 主要4答はキーワードのみで再現できるまで。再現できない回答は長すぎるか自分の言葉になっていない＝原稿側を修正
4. **声音層**: 自己紹介・志望動機を OJAD スズキクンに貼って韻律を確認し、社名・氏名の読みを固定（`native-delivery-pronunciation.md` §5-6）
5. 面接前72時間: evidence-ledger の数字確認 / 面接前7日以内: 企業ニュース再確認（D6）
